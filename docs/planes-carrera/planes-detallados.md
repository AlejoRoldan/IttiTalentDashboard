## 1. DESARROLLO DE SOFTWARE

### Ruta 1: Frontend Developer → Tech Lead Frontend → Engineering Manager
**Duración total**: 4-6 años | **Objetivo**: Liderazgo técnico y gestión de equipos frontend

#### Nivel 1: Junior Frontend Developer (0-2 años)

**Descripción del rol**: Desarrollador frontend en formación que trabaja en tareas bien definidas bajo supervisión directa, implementando interfaces de usuario funcionales y responsivas.

**Competencias clave**:
- HTML5, CSS3, JavaScript (ES6+)
- React o Angular (framework principal de Itti)
- Git y control de versiones colaborativo
- Responsive design y mobile-first
- Testing básico (Jest, React Testing Library)

**Hitos de desarrollo**:
- Completar 3 proyectos de interfaz de usuario de complejidad media
- Dominar el framework principal (React/Angular) y sus patrones
- Participar activamente en code reviews recibiendo feedback constructivo
- Resolver 50+ bugs y mejoras en código legacy

**Métricas de éxito**:
- Velocidad de entrega: 80% de tareas completadas en tiempo estimado
- Calidad de código: Máximo 2 bugs críticos por sprint
- Colaboración: Participación en 100% de code reviews asignadas

---

#### Nivel 2: Frontend Developer (2-4 años)

**Descripción del rol**: Desarrollador frontend autónomo capaz de diseñar e implementar features completas, trabajando con mínima supervisión y contribuyendo a decisiones técnicas del equipo.

**Competencias clave**:
- Arquitectura de componentes reutilizables
- State management (Redux, Context API, Zustand)
- Performance optimization (lazy loading, code splitting)
- Accesibilidad web (WCAG 2.1)
- Metodologías ágiles (Scrum, Kanban)

**Hitos de desarrollo**:
- Liderar desarrollo de 2 módulos completos end-to-end
- Mentoría activa a 1 desarrollador junior
- Implementar mejoras de performance que reduzcan tiempo de carga 30%
- Contribuir a la librería de componentes compartidos

**Métricas de éxito**:
- Autonomía: 90% de tareas completadas sin escalamiento
- Impacto: Mejoras medibles en métricas de usuario (load time, interactividad)
- Mentoría: Junior mentorado avanza a mid-level en 18 meses

---

#### Nivel 3: Senior Frontend Developer (4-6 años)

**Descripción del rol**: Referente técnico frontend que diseña arquitecturas escalables, define estándares de código y guía decisiones técnicas estratégicas del equipo.

**Competencias clave**:
- Diseño de arquitectura frontend escalable
- Micro-frontends y module federation
- CI/CD pipelines para frontend
- Liderazgo técnico y toma de decisiones
- Comunicación efectiva con stakeholders

**Hitos de desarrollo**:
- Diseñar arquitectura de aplicación completa (ej: core bancario web)
- Liderar equipo de 2-3 desarrolladores en proyecto crítico
- Presentar soluciones técnicas a dirección y stakeholders
- Establecer estándares de código y best practices del equipo

**Métricas de éxito**:
- Arquitectura: Sistema soporta 10x carga sin refactoring mayor
- Liderazgo: Equipo cumple 95% de sprints exitosamente
- Influencia: Propuestas técnicas aprobadas por dirección

---

#### Nivel 4: Tech Lead Frontend (6-8 años)

**Descripción del rol**: Líder técnico que define la visión tecnológica del área frontend, gestiona equipos de 5-8 personas y alinea la estrategia técnica con objetivos de negocio.

**Competencias clave**:
- Visión estratégica tecnológica frontend
- Gestión de equipos técnicos (5-8 personas)
- Roadmap tecnológico y planificación
- Presupuestos y gestión de recursos
- Innovación y adopción de tendencias

**Hitos de desarrollo**:
- Definir stack tecnológico frontend de Itti para próximos 3 años
- Implementar procesos de desarrollo que reduzcan time-to-market 40%
- Obtener certificaciones avanzadas (AWS Certified Developer, etc.)
- Liderar migración tecnológica crítica (ej: Angular → React)

**Métricas de éxito**:
- Estrategia: Roadmap cumplido en 90% de hitos
- Equipo: Retención de talento >85%, satisfacción >4/5
- Innovación: 2+ iniciativas de mejora implementadas por año

---

#### Nivel 5: Engineering Manager (8+ años)

**Descripción del rol**: Gerente de ingeniería que lidera múltiples equipos frontend/full-stack, define estrategia organizacional de desarrollo y gestiona presupuestos significativos.

**Competencias clave**:
- Gestión de múltiples equipos (20+ personas)
- Estrategia organizacional de ingeniería
- People management y desarrollo de talento
- OKRs y métricas de negocio
- Transformación digital y cambio organizacional

**Hitos de desarrollo**:
- Gestionar 3+ equipos con diferentes focos (productos, plataformas, innovación)
- Implementar programa de desarrollo de talento con resultados medibles
- Alinear estrategia técnica con objetivos de negocio (revenue, eficiencia)
- Participar en decisiones estratégicas de la Unidad Tecnológica

**Métricas de éxito**:
- Negocio: Contribución medible a revenue o reducción de costos
- Talento: Pipeline de líderes técnicos (2+ seniors → tech leads por año)
- Estrategia: Iniciativas alineadas con OKRs corporativos

### Ruta 2: Backend Developer → Software Architect → CTO
**Duración total**: 8-12 años | **Objetivo**: Liderazgo técnico corporativo

#### Nivel 1: Junior Backend Developer (0-2 años)

**Descripción del rol**: Desarrollador backend en formación que implementa funcionalidades bien definidas en sistemas existentes, trabajando bajo supervisión directa.

**Competencias clave**:
- Java, .NET o Node.js (stack de Itti)
- Bases de datos SQL (MySQL, Oracle)
- APIs RESTful y principios HTTP
- Principios SOLID y clean code
- Testing unitario (JUnit, NUnit, Jest)

**Hitos de desarrollo**:
- Desarrollar 5 APIs funcionales con documentación completa
- Optimizar consultas de base de datos reduciendo tiempo de respuesta 50%
- Implementar autenticación y autorización en 2 servicios
- Contribuir a documentación técnica del equipo

**Métricas de éxito**:
- Calidad: Cobertura de tests >70% en código nuevo
- Performance: APIs responden en <200ms percentil 95
- Colaboración: Feedback positivo en code reviews

---

#### Nivel 2: Backend Developer (2-4 años)

**Descripción del rol**: Desarrollador backend autónomo que diseña e implementa servicios completos, integrando múltiples sistemas y tomando decisiones técnicas fundamentadas.

**Competencias clave**:
- Arquitectura de microservicios
- Message queues (RabbitMQ, Kafka)
- Caching strategies (Redis, Memcached)
- Seguridad de aplicaciones (OWASP Top 10)
- Docker y contenedores

**Hitos de desarrollo**:
- Migrar módulo monolítico a microservicios
- Implementar sistema de caching que reduzca carga DB 60%
- Reducir tiempos de respuesta de APIs críticas 40%
- Mentoría a 1 junior backend developer

**Métricas de éxito**:
- Escalabilidad: Servicios soportan 3x carga sin degradación
- Autonomía: 85% de features entregadas sin escalamiento
- Impacto: Mejoras medibles en performance de sistemas

---

#### Nivel 3: Senior Backend Developer (4-7 años)

**Descripción del rol**: Referente técnico backend que diseña arquitecturas distribuidas, define patrones de integración y lidera decisiones técnicas de alto impacto.

**Competencias clave**:
- Arquitectura distribuida y event-driven
- Cloud computing (AWS/Azure) avanzado
- Escalabilidad y alta disponibilidad
- DevOps practices y CI/CD
- Liderazgo técnico y mentoría

**Hitos de desarrollo**:
- Diseñar sistema de alta disponibilidad (99.99% uptime)
- Liderar migración de core bancario a cloud
- Mentoría a equipo de 3-5 desarrolladores
- Definir estándares de arquitectura backend

**Métricas de éxito**:
- Confiabilidad: Uptime >99.9% en sistemas críticos
- Liderazgo: Equipo entrega proyectos en tiempo 90% de veces
- Arquitectura: Decisiones técnicas validadas en producción

---

#### Nivel 4: Software Architect (7-10 años)

**Descripción del rol**: Arquitecto de software que define la arquitectura empresarial, evalúa tecnologías estratégicas y asegura la coherencia técnica entre sistemas.

**Competencias clave**:
- Diseño de arquitectura empresarial

**Métricas de éxito**:

- Innovación: Adopción de 2+ tecnologías emergentes con ROI positivo

---

## 2. INFRAESTRUCTURA Y OPERACIONES

### Ruta: DevOps Engineer → SRE → VP of SRE
**Duración total**: 6-10 años | **Objetivo**: Liderazgo en confiabilidad y operaciones

#### Nivel 1: Junior DevOps Engineer (0-2 años)

**Descripción del rol**: Ingeniero DevOps en formación que automatiza procesos manuales, configura pipelines CI/CD básicos y aprende prácticas de infraestructura como código.

**Competencias clave**:
- Linux/Unix administration
- Scripting (Bash, Python)
- Git y control de versiones
- Docker y contenedores básicos
- Monitoreo básico (Prometheus, Grafana)

**Hitos de desarrollo**:
- Automatizar 5 procesos manuales repetitivos
- Configurar pipelines CI/CD para 3 aplicaciones
- Implementar monitoreo de aplicaciones con alertas
- Contribuir a documentación de runbooks

**Métricas de éxito**:
- Automatización: 80% de tareas manuales eliminadas
- CI/CD: Deployment time reducido 50%
- Monitoreo: 100% de aplicaciones con alertas configuradas

---

#### Nivel 2: DevOps Engineer (2-4 años)

**Descripción del rol**: Ingeniero DevOps autónomo que gestiona infraestructura cloud, implementa Kubernetes y optimiza pipelines de deployment.

**Competencias clave**:
- Kubernetes (orquestación de contenedores)
- Infrastructure as Code (Terraform, CloudFormation)
- Jenkins/GitLab CI avanzado
- Cloud platforms (AWS/Azure/GCP)
- DevSecOps y seguridad

**Hitos de desarrollo**:
- Migrar 10 aplicaciones a Kubernetes
- Reducir deployment time 60% con pipelines optimizados
- Implementar disaster recovery con RTO <1 hora
- Automatizar provisioning de infraestructura completa

**Métricas de éxito**:
- Eficiencia: Deployment frequency aumenta 5x
- Confiabilidad: Mean Time To Recovery (MTTR) <30 minutos
- Costo: Optimización de cloud costs 20%

---

#### Nivel 3: Senior DevOps Engineer (4-6 años)

**Descripción del rol**: Ingeniero DevOps senior que diseña arquitecturas cloud multi-región, implementa observability avanzada y lidera equipos técnicos.

**Competencias clave**:
- Arquitectura cloud multi-región
- Observability (Prometheus, Grafana, ELK, Jaeger)
- Chaos engineering (resilience testing)
- Cost optimization y FinOps
- Liderazgo de equipo DevOps

**Hitos de desarrollo**:
- Diseñar infraestructura multi-región con failover automático
- Reducir costos cloud 30% sin impactar performance
- Liderar equipo de 3-4 ingenieros DevOps
- Implementar chaos engineering en producción

**Métricas de éxito**:
- Confiabilidad: Uptime >99.95% en servicios críticos
- Costo: ROI positivo en optimizaciones (savings >100k USD/año)
- Liderazgo: Equipo entrega proyectos en tiempo 90%

---

#### Nivel 4: Site Reliability Engineer (SRE) (6-8 años)

**Descripción del rol**: Ingeniero SRE que define SLOs/SLIs, gestiona incidentes críticos y construye sistemas auto-reparables con enfoque en confiabilidad.

**Competencias clave**:
- SLOs, SLIs, SLAs (Service Level Objectives)
- Incident management y postmortems
- Capacity planning y forecasting
- Performance engineering
- Automation avanzada (self-healing systems)

**Hitos de desarrollo**:
- Lograr 99.99% uptime en core bancario
- Implementar SRE practices en toda la organización
- Reducir MTTR 50% con automation
- Definir error budgets y políticas de deployment

**Métricas de éxito**:
- Confiabilidad: 99.99% uptime sostenido 12 meses
- Eficiencia: Toil reducido 70% con automation
- Incidentes: MTTR <15 minutos, postmortems 100%

---

#### Nivel 5: VP of SRE (8+ años)

**Descripción del rol**: Vicepresidente de Site Reliability Engineering que define la estrategia de confiabilidad corporativa, gestiona equipos globales y lidera la cultura SRE.

**Competencias clave**:
- Estrategia de confiabilidad corporativa
- Gestión de múltiples equipos SRE/DevOps
- Presupuesto y recursos (millones USD)
- Cultura SRE organizacional
- Innovación en operaciones

**Hitos de desarrollo**:
- Definir estrategia de confiabilidad para todos los productos de Itti
- Construir equipo SRE de 20+ personas con alta retención
- Implementar cultura SRE con error budgets y blameless postmortems
- Posicionar a Itti como referente en confiabilidad en la región

**Métricas de éxito**:
- Confiabilidad: 99.99% uptime en todos los servicios críticos
- Eficiencia: Deployment frequency 10x, MTTR <10 minutos
- Cultura: Encuestas de satisfacción >4.5/5, retención >90%

## Metodología de Implementación

### Evaluación de Competencias

**Frecuencia**: Trimestral  
**Método**: 360° feedback (auto-evaluación, manager, peers, reports)  
**Escala**: 1-5 (1=Básico, 2=Intermedio, 3=Avanzado, 4=Experto, 5=Referente)

### Definición de Planes de Carrera

**Proceso**:
1. **Autoevaluación**: Empleado identifica intereses y objetivos
2. **Evaluación manager**: Manager valida competencias actuales
3. **Co-creación**: Empleado y manager definen plan conjunto
4. **Revisión RRHH**: Validación de viabilidad y recursos
5. **Aprobación**: Dirección aprueba planes de alto impacto

### Seguimiento y Ajustes

**Frecuencia**: Mensual (1:1 con manager)  
**Revisión formal**: Trimestral (evaluación de progreso)  
**Ajustes**: Según cambios en objetivos personales o necesidades de negocio

### Indicadores de Éxito

**Individuales**:
- Progreso en competencias (delta trimestral)
- Cumplimiento de hitos (% completado)
- Satisfacción con plan de carrera (encuesta)

**Organizacionales**:
- Retención de talento (>85% en roles críticos)
- Promociones internas (>70% de posiciones senior)
- Time-to-productivity de nuevos roles (<3 meses)
